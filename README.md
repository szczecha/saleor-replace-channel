# Saleor Channel Migration Script

Saleor does not allow changing a channel's currency after creation. This script migrates all product, variant, and shipping method listings from one channel to another — preserving all existing settings — then removes the old channel assignment.

## What it does

1. Fetches all products assigned to the old channel (up to 50)
2. For each product: adds it to the new channel with the same `isPublished`, `publishedAt`, `isAvailableForPurchase`, `availableForPurchaseDate`, and `visibleInListings` settings, then removes the old channel
3. For each product's variants: adds them to the new channel with the same price (same numeric amount), then removes the old channel — using a bulk update per product
4. Fetches all shipping zones assigned to the old channel (up to 10)
5. For each shipping method: adds it to the new channel with the same price, then removes the old channel

Errors are logged per item and the script continues processing the rest.

## Prerequisites

- Python 3.8+
- A Saleor app token with `MANAGE_PRODUCTS` and `MANAGE_SHIPPING` permissions
- The new channel must already exist in Saleor before running the script, it must be assign to shipping zones to update shipping methods pricing.

## Setup

```bash
pip install -r requirements.txt
cp .env.example .env
```

Edit `.env` with your values:

```env
SALEOR_API_URL=https://your-store.saleor.cloud/graphql/
SALEOR_AUTH_TOKEN=your-app-token-here
OLD_CHANNEL_SLUG=default-channel
NEW_CHANNEL_SLUG=channel-lkr
```

## Usage

**Dry run** — preview all planned operations without executing any mutations:

```bash
python migrate_channel.py --dry-run
```

**Live run** — migrate everything:

```bash
python migrate_channel.py
```

**Migrate only products and variants:**

```bash
python migrate_channel.py --products
python migrate_channel.py --products --dry-run
```

**Migrate only shipping methods:**

```bash
python migrate_channel.py --shipping
python migrate_channel.py --shipping --dry-run
```
